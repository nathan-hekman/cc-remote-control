# cc-imessage-remote-control — setup wizard

Drive this wizard interactively. Each step has a clear ask, a wait point,
and a verification. Do not run all steps in one shot. Do not assume any
answer.

## Pre-flight (silent — do this before talking to the user)

1. Resolve the absolute install path:
   ```bash
   echo "${CLAUDE_PLUGIN_ROOT}"
   ```
   Save this as `INSTALL_PATH`. The Shortcut will need this exact path.
2. Resolve the config file path:
   - `CONFIG_DIR="${CLAUDE_CONFIG_DIR:-$HOME/.claude}"`
   - `CONFIG_FILE="$CONFIG_DIR/.cc-imessage-env"`
3. Check if `CONFIG_FILE` already exists. If so, treat this as a re-run
   and offer to update rather than overwrite.
4. Check that the `claude` CLI is on PATH (`command -v claude`). If
   missing, stop and tell the user to install Claude Code first.

## Step 1 — Collect required config

Ask the user three things (use AskUserQuestion for the first, plain text
for the rest):

**Q1 (multi-select=false, two options):**
- "What phone number should I reply to?" — option A: "Same number I'll
  text *from* (typical case)", option B: "A different number".
- If A, ask plainly for the number. If B, ask for both the *trigger*
  number (sender filter) and the *reply* number separately.

**Q2 — projects root.** Ask: "Where do your code projects live? Default
is `$HOME/Documents`. Press enter to accept, or paste a different path."
Validate the path exists (`[ -d "$path" ]`).

**Q3 — extra dirs (optional).** Ask: "Do you keep some projects nested
one level deeper (e.g. `~/Documents/Other Projects/`)? If yes, paste the
parent dir; the router will scan one level into it too. Otherwise press
enter."

## Step 2 — Write the config file

Write the values to `$CONFIG_FILE` with proper quoting:

```bash
cat > "$CONFIG_FILE" <<EOF
# cc-imessage-remote-control config
# Generated by /cc-imessage setup on $(date -Iseconds)
IMESSAGE_TARGET="<phone>"
IMESSAGE_PREFIX="[CR]"
ROUTER_MODEL="claude-haiku-4-5-20251001"
PROJECTS_ROOT="<projects-root>"
PROJECTS_ROOT_EXTRA="<extra-or-empty>"
PROJECTS_EXCLUDE=""
EOF
chmod 600 "$CONFIG_FILE"
```

Confirm to the user: "Wrote config to `~/.claude/.cc-imessage-env`.
Phone number masked: `+1 *** *** ####`."

## Step 3 — Sanity check the project list

Run:
```bash
bash "${CLAUDE_PLUGIN_ROOT}/bin/build_project_list.sh"
```

Show the user the first 15 lines. Ask: "Do you see the projects you
want? Anything missing, or anything that shouldn't be there?"

If they want to exclude something, edit `$CONFIG_FILE` to add a
`PROJECTS_EXCLUDE` entry and re-run.

## Step 4 — Print the Shortcuts shell-script line

This is the **critical** moment. Print this to the chat verbatim with a
clear "copy this exact line" instruction:

```
"$INSTALL_PATH/bin/claude-router.sh" "$1"
```

(With `$INSTALL_PATH` already substituted to the resolved absolute path.)

Tell the user: "In a moment we'll open Shortcuts.app. You'll create one
shortcut + one automation. The shortcut needs that exact line above as
its `Run Shell Script` action. Pass Input must be set to **'as
arguments'** — that's the part that makes `$1` work."

## Step 5 — Open Shortcuts.app

Ask: "Ready to open Shortcuts.app? (yes / not yet)"

If yes, run:
```bash
osascript -e 'tell application "Shortcuts" to activate'
```

Walk them through, in order:

1. **All Shortcuts** sidebar → **+** → create new shortcut.
2. Drag in a **Run Shell Script** action (search the actions panel).
3. Shell: `zsh` (or `/bin/bash`).
4. Pass Input: **as arguments** (NOT "to stdin").
5. Run as Administrator: **off**.
6. Script: paste the exact line printed in Step 4.
7. Rename the shortcut to **"Run claude launcher"**.
8. `⌘W` to close.
9. Switch to the **Automation** tab in the sidebar.
10. **+** → New Automation → pick the **Message** trigger.
11. Sender: themselves (their own contact card — this is the safety
    gate that stops anyone else triggering it).
12. Message Contains: `claude` (lowercase — case-insensitive).
13. **Run Immediately** selected.
14. Next → delete any default actions → add **Run Shortcut** →
    `Run claude launcher` → make sure Pass Input is **Shortcut Input**.
15. Done.

Reference screenshots are in the repo at `docs/screenshots/01-*.jpeg`
through `03-*.jpeg` — link them inline as you walk through.

## Step 6 — Self-test

Run the router with a test phrase that doesn't actually open a Terminal
window. We do this by passing a non-matching phrase so we exercise the
infer + reply path without spawning a session:

```bash
bash "${CLAUDE_PLUGIN_ROOT}/bin/claude-router.sh" "Claude" 2>&1 | tail -5
```

This should send the user an iMessage that looks like
`[CR] Which project? Try one of: <list>`.

Ask: "Did you receive an iMessage from your Mac (prefixed `[CR]`)?"

- **Yes** → setup is done. Tell them to text "Claude <project>" from
  iPhone next.
- **No** → diagnose:
  - Check `tail -20 ~/.claude/.cc-imessage-logs/router.log` for the
    last run's errors.
  - If Messages.app permission popup appeared, accept it.
  - If `IMESSAGE_TARGET` looks wrong, edit `$CONFIG_FILE`.
  - If the macOS Automation permission for Shortcuts → Messages isn't
    granted, point them to System Settings → Privacy & Security →
    Automation.

## Step 7 — Wrap up

Print these one-liners they should bookmark:

- `/cc-imessage status` — see config + project list + recent logs
- `/cc-imessage test` — run the router locally (no iMessage round-trip)
- `/cc-imessage tail` — last 20 log lines

Plus the live docs URL: https://nathan-hekman.github.io/cc-imessage-remote-control/

## Style

- One step at a time. Wait for the user between steps.
- Surface the actual paths and values you resolved — don't be vague.
- If something errors, show the user the exact line from the log, not a
  paraphrase.
- Don't auto-overwrite an existing config without asking.
- Mask the phone number when echoing config back to the chat
  (`+1 *** *** ####`).
